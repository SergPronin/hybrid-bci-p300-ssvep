## О проекте

Система визуальной стимуляции и мониторинга для гибридного BCI-интерфейса (P300). В основе — **событийно-ориентированное приложение для обработки потоковых данных в реальном времени** (Event-driven real-time stream processing).

Компоненты системы работают независимо и обмениваются данными через единый протокол синхронизации. Стимуляция генерирует события, мониторинг подписывается на потоки и обрабатывает их асинхронно с минимальной задержкой.

## Архитектурные решения

### Модульная структура

Проект разделён на три модуля:

| Модуль | Назначение |
|--------|------------|
| **core/** | Бизнес-логика: модели плиток, сетка, контроллер стимуляции, отправка маркеров |
| **gui/** | Визуализация и взаимодействие с пользователем |
| **scripts/** | Валидация аппаратуры ЭЭГ (hardware_validation) |

### Publisher–Subscriber через LSL

Для передачи событий между независимыми компонентами используется паттерн **Publisher-Subscriber** через протокол **Lab Streaming Layer (LSL)**. Это позволяет:

- **Развязать компоненты** — стимуляция, ЭЭГ-источник и скрипт валидации работают как отдельные процессы
- **Масштабировать систему** — можно добавлять новые подписчики без изменения издателей
- **Синхронизировать потоки** — LSL обеспечивает временную привязку маркеров и сигналов

Приложение стимуляции публикует маркеры (`BCI_StimMarkers`); скрипт валидации подписывается на поток ЭЭГ. Аналогия с брокерами сообщений (Kafka, RabbitMQ): издатель не знает о подписчиках, подписчики получают события по мере поступления.

### Асинхронная обработка и буферизация

- **Ring buffers / deque** — скользящий буфер ЭЭГ хранит последние N секунд данных для извлечения эпох по маркерам
- **Queues** — очередь маркеров для обработки в порядке поступления при накоплении буфера
- **QTimer** — событийно-ориентированное обновление UI и чтение LSL без блокировки главного потока
- **pull_chunk / pull_sample** — неблокирующее чтение потоков с таймаутом

## Стек технологий

| Технология | Назначение |
|------------|------------|
| **Python 3.10** | Язык реализации |
| **PyQt5** | Event-loop и UI для дашборда |
| **pyqtgraph** | Рендеринг графиков с высокой частотой кадров (~30 FPS) |
| **SciPy** | Цифровая фильтрация сигналов (Butterworth bandpass 0.5–10 Гц) |
| **pylsl** | Lab Streaming Layer — протокол потоковой передачи данных |
| **PsychoPy** | Визуальная стимуляция (основное приложение) |
| **NumPy** | Обработка массивов и вычисления |

## Установка и запуск

### Требования

- **Python 3.10** (PsychoPy не поддерживает Python 3.11+)
- macOS / Linux / Windows

### 1. Клонирование и виртуальное окружение

```bash
git clone <repository-url>
cd hybrid-bci-p300-ssvep

python3.10 -m venv .venv
source .venv/bin/activate   # macOS/Linux
# .venv\Scripts\activate    # Windows
```

### 2. Зависимости

```bash
pip install -r requirements.txt
```

### 3. Запуск модулей

**Приложение стимуляции** (основное):

```bash
python -m app.main
```

Управление: **START** / **STOP**, слайдеры интервала и длительности вспышки, **Escape** — выход.

**Валидация аппаратуры ЭЭГ** (визуализация каналов, масштабирование, сохранение данных):

```bash
python scripts/hardware_validation.py
```

Скрипт можно запускать без потока; в окне кнопка «Поиск потока» подключается к нейроспектру или симулятору по LSL.

## Протокол LSL

- **Маркеры:** `BCI_StimMarkers`, тип `Markers`, формат `tile_id|event` (например `3|on`, `3|off`)
- **ЭЭГ:** тип `EEG` (или `Signal` в зависимости от оборудования)
