# BCI Monitor Dashboard - Профессиональный интерфейс мониторинга P300

## Описание

Профессиональный дашборд для мониторинга P300-сигналов в реальном времени. Интерфейс состоит из трех главных зон:

### ЗОНА 1: Непрерывный сигнал (Верхняя половина окна)
- **Бегущая волна ЭЭГ**: Отображает последние 10 секунд ЭЭГ-сигнала в реальном времени
- **Маркеры стимулов**: Вертикальные желтые линии с подписью номера плитки в момент каждой вспышки
- **Переключатель фильтра**: Чекбокс для переключения между сырым и отфильтрованным (0.5-10 Гц) сигналом
- **Масштаб Y**: Ползунок для изменения масштаба по оси Y (10-200 мкВ)

### ЗОНА 2: Сетка когерентного накопления P300 (Нижняя левая часть)
- **9 графиков 3x3**: Показывают реакцию мозга на каждую конкретную плитку
- **Единая ось Y**: Все графики используют одинаковый диапазон (-20 до +20 мкВ) для корректного сравнения
- **Маркер нуля**: Яркая вертикальная красная линия на отметке 0.0 сек (момент стимула)
- **Окно поиска P300**: Светло-зеленая полупрозрачная заливка в диапазоне 0.25-0.55 сек
- **Butterfly Plot**: Опциональная функция для отображения всех сырых эпох тонкими линиями

### ЗОНА 3: Панель управления и статуса (Нижняя правая часть)
- **Метрики**:
  - Всего эпох: Количество обработанных вспышек
  - Качество сигнала: Дисперсия сигнала с индикацией артефактов (зеленый/красный)
  - Результат классификатора: Определение плитки с максимальным P300
- **Кнопки управления**:
  - **Start Processing**: Подключиться к LSL и начать обработку
  - **Reset Buffers**: Сбросить все накопленные эпохи
  - **Stop & Save**: Остановить сбор и сохранить результаты

## Запуск

```bash
python scripts/bci_monitor_dashboard.py
```

## Требования

Перед запуском убедитесь, что:
1. Запущено приложение стимуляции (`python -m app.main`)
2. Запущен поток ЭЭГ (например, Нейроспектр)
3. Оба потока транслируются через LSL

## Технические детали

### Используемые технологии
- **PyQt5**: Для создания интерфейса и управления элементами
- **PyQtGraph**: Для высокопроизводительной визуализации графиков (OpenGL)

### Параметры обработки
- **Эпоха**: -0.2 до +0.8 сек относительно стимула
- **Фильтр**: Bandpass 0.5-10 Гц (Butterworth 4-го порядка)
- **Baseline correction**: Вычитание среднего значения в период -0.2 до 0.0 сек
- **Окно поиска P300**: 0.25-0.55 сек после стимула
- **Порог качества сигнала**: Дисперсия > 15 мкВ считается артефактом

### Классификатор
Простой классификатор определяет плитку с максимальной амплитудой P300 в окне поиска. Результат отображается только если амплитуда превышает порог 2 мкВ.

## Сохранение результатов

При нажатии кнопки "Stop & Save" результаты сохраняются в файл:
```
results/eeg_epochs_YYYYMMDD_HHMMSS.pkl
```

Файл содержит:
- Все накопленные эпохи для каждой плитки
- Усреднённые эпохи
- Параметры обработки
- Статистику по количеству эпох

## Особенности реализации

1. **Единая ось Y для всех графиков P300**: Критически важно для корректного визуального сравнения амплитуд между плитками
2. **Непрерывная фильтрация**: Фильтр применяется к каждому новому сэмплу с сохранением состояния
3. **Реальное время**: Обновление графиков происходит с частотой ~30 FPS
4. **Детекция артефактов**: Автоматическое определение артефактов по дисперсии сигнала

## Устранение неполадок

### Потоки LSL не найдены
- Убедитесь, что приложение стимуляции запущено и нажата кнопка START
- Проверьте, что ЭЭГ оборудование транслирует данные через LSL
- Используйте `python scripts/lsl_listen.py` для проверки доступных потоков

### Графики не обновляются
- Убедитесь, что нажата кнопка "Start Processing"
- Проверьте, что данные поступают из LSL потоков

### Низкое качество сигнала
- Проверьте контакт электродов
- Убедитесь, что испытуемый сидит спокойно (без движений головой, морганий)
- Проверьте заземление оборудования
