# CCA-метод для выбора плиток с использованием P300-эталона

Реализован новый метод выбора плиток на основе **каноническом корреляции анализе (CCA)** с использованием P300-эталона, построенного из реальных калибровочных сессий.

## Принцип работы

### 1. Построение P300-эталона (калибровка)

В этапе калибровки накапливаются эпохи из нескольких сессий (≈ 10 разных блоков с разными плитками):

```bash
python scripts/build_p300_template.py /path/to/calibration/data/ --output my_p300_template.npz
```

Что происходит:
- Загружаются все CSV файлы с реальными P300-ответами
- Эпохи накапливаются для каждого типа стимула
- Вычисляется среднее P300 (эталон)
- Применяется базовая коррекция
- Результат сохраняется в файл NPZ

На выходе:
- `my_p300_template.npz` — главный эталон (средний P300)
- `my_p300_template.json` — метаинформация (количество стимулов, форма, и т.д.)

### 2. Использование CCA во время теста

#### В GUI (онлайн или офлайн)

1. Загрузить эталон: нажмите **"Загрузить P300-эталон для CCA"**
2. Выберите режим: комбобокс → **"CCA (каноническая корреляция с P300-шаблоном)"**
3. Работайте как обычно — CCA будет использовать загруженный эталон

#### Программно

```python
from p300_analysis.p300_template import load_p300_template
from p300_analysis.erp_compute import compute_winner_metrics

# Загрузить эталон
template, time_ms, stim_templates = load_p300_template("my_p300_template.npz")

# Использовать при вычислении метрик
winner_idx, mode_used, debug = compute_winner_metrics(
    stim_keys=stim_keys,
    raw_averaged=raw_averaged,
    corrected=corrected,
    time_ms=time_ms,
    window_x_ms=100,
    window_y_ms=600,
    winner_mode="cca",
    p300_template=template,  # ← Передать эталон
)
```

## Технические детали

### Алгоритм CCA

Для каждого стимула:
1. Входные данные: EEG сигнал в окне [X, Y]
2. Эталон: P300-подобный шаблон (из калибровки)
3. Вычисляется каноническая корреляция между сигналом и эталоном
4. Выходная оценка: корреляция ∈ [0, 1]
5. **Победитель = стимул с максимальной корреляцией**

### Регуляризация и устойчивость

- Обработка нулевой дисперсии
- Защита от NaN/Inf
- Интерполяция если окна не совпадают по размеру
- Fallback на синтетический гаусс если эталон не загружен

## Сравнение методов

| Метод | Эталон | Описание |
|-------|--------|---------|
| **AUC** | - | Интегрирование по модулю |corrected| |
| **Signed Mean** | - | Среднее значение в окне |
| **MSI-like** | Синтетический гаусс | Комбинация энергии + шаблон |
| **CCA** | Реальный P300 (калибровка) | Каноническая корреляция с эталоном |

## Примеры файлов структуры

### Калибровочные данные

```
calibration/
  ├─ eeg_2025-12-01_10-30-00.csv
  ├─ eeg_2025-12-01_11-15-30.csv
  ├─ eeg_2025-12-01_14-20-45.csv
  └─ ...
```

### Построенный эталон

```
p300_template.npz  ← Содержит:
  ├─ template      (epoch_len,) — главный эталон
  ├─ time_ms       (epoch_len,) — временная шкала
  ├─ stim_0X...    (epoch_len,) — эталон для каждого стимула
  └─ ...

p300_template.json ← Метаинформация
  {
    "template_shape": [150],
    "time_ms_shape": [150],
    "stim_keys": ["стимул_0", "стимул_1", ...],
    "source": "build_p300_template"
  }
```

## Рекомендации

1. **Минимум калибровочных сессий**: 10 блоков с разными плитками
2. **Качество калибровки**: используйте чистые данные без артефактов
3. **Параметры**: используйте те же окно [X, Y] и baseline_ms что и при калибровке
4. **Проверка**: сравните результаты CCA с MSI-like на том же наборе данных

## Возможные ошибки

| Ошибка | Решение |
|--------|---------|
| Template not found | Сначала запустите `build_p300_template.py` |
| Wrong template shape | Проверьте параметры окна [X, Y] |
| No stim_keys in template | Калибровочные файлы должны быть непустыми |
| CCA correlation is NaN | Проверьте качество данных, может быть нулевая дисперсия |
